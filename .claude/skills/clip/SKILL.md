# clip

会話の結果や過程をまとめてクリップボードにコピーするスキル。

## 引数の解釈

`$ARGUMENTS` に応じて出力内容を決定する:

- **引数あり** → 指示に従って内容を生成する
  - 例: `この議論の結論` → 議論の結論を整理して出力
  - 例: `コードの差分` → 変更内容をまとめて出力
  - 例: `レビュー結果` → レビュー指摘をまとめて出力
- **引数なし** → セッションで行った作業の要約を生成する

## 出力ルール

### デフォルト（引数なし）: 作業要約

セッション中に行ったことを以下の構造で要約する:

```
## 作業サマリー

### やったこと
- [完了したタスクを箇条書き]

### 変更ファイル
- [変更したファイルのリスト（あれば）]

### 未完了・次のアクション
- [残タスクや次にやるべきこと（あれば）]
```

### 議論・相談の場合

会話が議論や設計相談だった場合は、より詳細に:

```
## 議論サマリー

### テーマ
[議論のテーマ]

### 論点と結論
1. [論点]: [結論]
2. [論点]: [結論]

### 決定事項
- [決まったこと]

### 未決事項・次のアクション
- [持ち越し事項]
```

### 引数ありの場合

指示に従い、適切な形式で内容を生成する。形式の指定がなければ Markdown で出力する。

## 実行手順

1. 引数と会話の文脈から、出力すべき内容を判断する
2. 内容をMarkdown形式で生成する
3. **必ず quoted heredoc でコピーする**（`echo` はクォートや `$` の展開事故が起きるため使わない）:

```bash
pbcopy <<'CLIP'
<生成した内容>
CLIP
```

4. コピー完了をユーザーに通知し、内容のプレビュー（冒頭数行）を表示する

## 注意事項

- macOS の `pbcopy` を使用する（Linux の場合は `xclip -selection clipboard`）
- 出力は簡潔に。冗長な説明は省き、共有に適した形式にする
- シークレット（APIキー、パスワード等）が含まれないよう注意する
- 会話内に直接出力せず、クリップボードのみに出力する
